## What shipped

### Navigation & Explore

- Renamed **Browse → Explore**; compass icon; mobile header is **labeled** (icon + “Explore”), not icon-only (P0 option 2 — tabs unchanged).
- Explore drawer: tabs **Stores | Aisles**; title follows active tab; **default Stores**.
- **All categories** (grid icon) opens **Aisles** only.
- Home ladder: Hero search → **Shop by store** (`#stores`) → Categories (`#categories`) → Deals → Featured.
- Desktop **See all** on stores still opens Explore → Stores; redundant mobile “Explore all stores” pill removed earlier and not restored.
- Mobile store/category pages: `BrowseSearchLink` → `/search` (marked `"use client"` for reicon).

### Save loop (P4)

- Add / create-list success toast includes **View list** action.
- List detail: **Add more** → Search or Explore; empty list also offers Explore stores.
- `deleteList` no longer auto-reseeds “Weekly shop” — true empty lists UI works.

### Back & hash (P5 + bugfix)

- `lib/nav-parent.ts` — forced parents for lists, profile subpages, store, category, legal.
- `HashScroll` — soft-nav to `/#stores` / `/#categories` actually scrolls.
- `NavigationHistory` — session path stack so Back uses `history.back()` only after in-app hops; deep links use safe fallback (e.g. product → `/search`).

### Content & chrome

- Desktop `SiteFooter`; pages under `/about`, `/faq`, `/privacy`, `/terms` (`legal-page.tsx`).
- Profile: utility links + feedback dialog; social pills with SVG brand icons (`social-icons.tsx`).
- Icons: **lucide → reicon-react**; category map in `category-icons.tsx` (client-safe).
- Returning visitor hero: no `localStorage` in `useState` init (hydration-safe `useEffect`).

### Visual

- Softened `.glass-nav` frost.
- Retailer logos fill circles via `object-cover` (home, Explore, store page, product card, **price comparison**).
- Add-to-list **+**: solid `bg-zinc-100` + ring (visible on white product images).

### Product compare / search polish (late session)

- Tied lowest prices: **all** stores at the min price get the Lowest badge (not only the first).
- Removed “Lay’s chips” from search capsule sample queries.

### Bug fix pass (all addressed)

| Issue | Fix |
|-------|-----|
| Hash parents don’t scroll | `HashScroll` |
| Search capsule ignores new `q` | sync `useEffect` on `initialQuery` |
| Product Back leaves site | path stack + parent fallback |
| List remove lag / race | optimistic filter + AbortSignal |
| Empty lists unreachable | stop reseed in `deleteList` |
| Explore sheet side flicker | `useMediaQuery` → `useSyncExternalStore` |
| Barcode confirm no abort / bad HTTP | AbortController + `res.ok` + toast |
| Toaster + missing ThemeProvider | hardcode `theme="light"` |

2. **Explore = Stores default; All categories = Aisles.**
3. Dialogs/sheets: bubble rules in `.cursor/rules/dialogs.mdc`.
4. Don’t restore a redundant full-width “Explore all stores” mobile pill.

## How to verify quickly

```bash
cd frontend && npm run dev
```

1. Mobile: header shows **Explore** label → Opens Stores tab.  
2. Home: stores section before categories; All categories → Aisles.  
3. Add product to list → toast **View list**.  
4. Store page Back → home scrolled to stores.  
5. Search empty-state pill → capsule shows same `q`.  
6. Delete last list → empty state (not auto Weekly shop).  
7. Barcode fail / close mid-lookup → no surprise navigation.
