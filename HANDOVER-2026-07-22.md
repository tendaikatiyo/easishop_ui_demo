## Conventions to preserve

1. Signup stays short. Extra DB fields (username, last name, marketing) belong on Profile.
2. Phone on auth = WhatsApp for alerts only (`intent=alert`).
3. Marketing opt-in defaults **off** and is not on the signup form.
4. Auth is never a welcome gate before search/compare.
5. Auth visuals: bottom-left orb only. No green wash, no green input fills, no doodles on `/signup` · `/signin`.

## How to verify

```bash
cd frontend && npm run dev
```

2. `/signup?intent=alert&next=/` — WhatsApp required with country code.  
3. `/signin` — Google, Email, Password, Forgot password.
