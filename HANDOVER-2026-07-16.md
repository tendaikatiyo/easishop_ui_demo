### Conversion onboarding

Spec: [`ux-onboarding-conversion.md`](./ux-onboarding-conversion.md).

| Layer | Behaviour |
|-------|-----------|
| **1 Welcome** | First home visit: Search / Deals / Skip + optional Create account · Sign in |
| **2 Aha tip** | Once on PDP with ≥2 available prices |
| **3a List** | After first add-to-list: guest → signup/signin soft prompt; signed-in → optional name |
| **3b Alert** | Guest → `/signup?intent=alert`; signed-in → email/phone (marketing opt-in **off**) |

Replay: footer **Onboarding** (desktop) · Profile **Onboarding** (mobile) → `replayOnboarding()` (clears flags + `signedIn: false`).

### Demo auth (local only)

- `lib/auth.ts` + `DemoUser.signedIn` — no real passwords / backend session.
- Routes: `/signup`, `/signin` (`?next=` · `?intent=`).
- **Not** a welcome gate — optional links + intent prompts (wise for conversion; matches production “lists need accounts”).
- Seeded demo user stays `signedIn: true`; replay / delete account leave guest.

### Auth page chrome

On `/signin` · `/signup`, `AppShell` uses a minimal layout:

- **Hidden:** search capsule, desktop nav, Explore, footer, mobile bottom tabs  
- **Shown:** Back · EasiShop logo → `/` · **Back to shop**  
- Form: Continue without an account → home
