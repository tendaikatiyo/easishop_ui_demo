## What shipped

### WhatsApp price alert

- **New helper** `lib/whatsapp.ts`:
  - `normalizeWhatsAppInput` — strips spaces/dashes/parens, keeps leading `+`, converts `00…` → `+…`.
  - `isValidWhatsAppNumber` — E.164-style `^\+[1-9]\d{7,14}$` (country code can't start with 0).
  - `whatsAppValidationError` — friendly messages: empty → "Enter your WhatsApp number"; local-looking (no `+`/`00`) → "Include a country code, e.g. +27 82 123 4567"; otherwise → "Use international format, e.g. +27821234567".
- **New icon** `WhatsAppIcon` in `components/layout/social-icons.tsx`.
- **Rewrote** `components/onboarding/price-alert-prompt.tsx`:
  - Button: `WhatsAppIcon` + "Alert me on WhatsApp" (was `Bell` + "Alert me when price drops").
  - Dialog: single WhatsApp number input (email field + marketing checkbox removed). Inline error under the field, error toast on invalid save, `aria-invalid` / `aria-describedby` wired.
  - On save: stores normalized number in `user.phone`, enables the `priceAlerts` entry, tracks `onboard_alert_prompt_accepted` with `via: "whatsapp"`.
  - Guest flow unchanged: not signed in → `/signup?next=…&intent=alert`.
  - Enabled state copy: "WhatsApp price alert on — manage under Profile."

### Dev hot-reload fix

- Dev log showed cross-origin HMR **blocked** from `192.168.0.37` (testing on another device → stale bundle, which is why the "dialog didn't pop up"; a hard refresh loaded the new code).
- Added `allowedDevOrigins: ["192.168.0.37"]` to `frontend/next.config.ts` so hot reload works from LAN devices. **Restart `npm run dev`** for it to take effect.

### Verification

- Drove the running dev server with headless Chrome (temporary diagnostic script, since removed): clicking "Alert me on WhatsApp" opens the dialog with overlay, no JS exceptions, `onboard_alert_prompt_shown` fires. Confirmed working.

## Conventions to preserve

1. Price alerts are **WhatsApp-only** now — do not reintroduce email/marketing into this dialog.
2. WhatsApp numbers must be **E.164** (leading `+`, 8–15 digits, country code ≠ 0). Reuse `lib/whatsapp.ts` helpers rather than re-implementing.
3. Auth-at-intent still applies: guests go to `/signup?intent=alert` before capturing a number.
4. Auth is never required before search/compare.
